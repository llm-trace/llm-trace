# Scoring API

Scores are evaluation metrics that you attach to a trace or a specific observation. Use them for quality evaluation, human feedback, A/B testing, or LLM-as-judge pipelines.

## Quick usage

```python
from llm_trace import score

score("quality", 0.92)              # numeric — attaches to current trace
score("passed_guardrail", True)     # boolean
score("category", "helpful")        # categorical
```

When called inside an `@observe()` function, `score()` automatically attaches to the active trace. No need to pass `trace_id`.

## Parameters

```python
score(
    name="quality",                   # score identifier
    value=0.92,                       # float, bool, or str
    trace_id=None,                    # auto-detected from context if inside @observe()
    observation_id=None,              # attach to a specific observation instead of the trace
    source="api",                     # "api" | "human" | "llm_judge" | "user_feedback"
    comment="Looks correct",          # optional explanation
)
```

### `name`
A string key identifying what is being scored. Use consistent names across traces to enable filtering and comparison in the dashboard (e.g. `"quality"`, `"groundedness"`, `"safety"`, `"user_feedback"`).

### `value`
Type is inferred automatically:

| Python type | Score type |
|-------------|------------|
| `float` / `int` | NUMERIC |
| `bool` | BOOLEAN |
| `str` | CATEGORICAL |

### `source`

| Value | Meaning |
|-------|---------|
| `"api"` | Set programmatically (default) |
| `"human"` | Annotated by a human reviewer |
| `"llm_judge"` | Generated by an LLM evaluation pipeline |
| `"user_feedback"` | Submitted by the end user (e.g. thumbs up/down) |

## Scoring inside @observe()

```python
@observe()
def pipeline(query: str) -> str:
    result = generate(query)
    score("quality", 0.88)               # attaches to this trace
    score("response_length", len(result)) # another metric
    return result
```

## Scoring after the fact

```python
from llm_trace.langchain import CallbackHandler
from llm_trace import score

handler = CallbackHandler(trace_name="chat")
result = graph.invoke(input, config={"callbacks": [handler]})
handler.flush()

trace_id = handler.get_trace_id()
score("user_feedback", 1, trace_id=trace_id, source="user_feedback")
score("grounded", True, trace_id=trace_id, source="llm_judge")
```

## Scoring a specific observation

```python
from llm_trace import tracer, score

@observe()
def pipeline(query: str) -> str:
    obs = tracer.current_observation
    result = generate(query)
    score("latency_ok", True, observation_id=obs.id)
    return result
```

## LLM-as-judge pattern

```python
@observe()
def evaluate(trace_id: str, response: str) -> None:
    prompt = f"Rate the quality of this response from 0 to 1:\n\n{response}"
    rating = float(openai_client.chat.completions.create(...).choices[0].message.content)
    score("quality", rating, trace_id=trace_id, source="llm_judge")
```

## User feedback endpoint (FastAPI example)

```python
@app.post("/api/feedback")
def feedback(trace_id: str, rating: int):
    # rating: 1 = thumbs up, -1 = thumbs down
    score("user_feedback", rating, trace_id=trace_id, source="user_feedback")
    return {"ok": True}
```

## Viewing scores

Scores appear in the dashboard on the trace detail page and in the CLI:

```bash
llm-trace show <trace-id>
# Shows all scores attached to the trace
```
